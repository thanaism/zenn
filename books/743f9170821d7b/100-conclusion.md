## さいごに

AAの10年を一本の線で並べると、こうなります。

| 提案     | 時期               | 位置づけ                                                       |
| -------- | ------------------ | -------------------------------------------------------------- |
| EIP-86   | 〜2017             | ネイティブAAの初期ビジョン・プロトコル改変が重く頓挫           |
| EIP-2938 | 2020               | 簡略化を試みるも頓挫・mempool担保の発想は4337へ                |
| EIP-3074 | 2020               | EOA拡張の別系譜・AUTH/AUTHCALLは採用されず、7702へ引き継がれる |
| ERC-4337 | 2021               | プロトコルを変えずにコントラクトレイヤーで実現・実用化         |
| EIP-7702 | 2025（Pectra）     | プロトコル改変への復帰、その第一歩                             |
| EIP-8141 | 2026（Hegotá予定） | ネイティブAAをプロトコルにenshrineする提案                     |

AAと聞くと新しめな印象がありますが、実はEthereum初期から続くネイティブAAという流れの一部だということを説明してきました。
そして、その系譜全体を「任意コード実行へのDoS対策」という技術的制約が支配しているわけです。

AAという技術を、その理念と技術的制約から整理する日本語記事は私の探した限りでは存在していなかったため、本書での整理がみなさんの理解の一助となれば幸いです。
